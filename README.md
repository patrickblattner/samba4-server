# samba4-server

A fully functional **Samba 4 Active Directory Domain Controller** running in Docker — for lab and development use.

> **Not for production.** This is a lab DC for developing and testing applications that need LDAP, Kerberos, SMB, or Active Directory authentication — without installing Windows Server.

## What you get

A single `./setup.sh` command gives you a complete AD Domain Controller with:

| Service | Port(s) | Description |
|---------|---------|-------------|
| **DNS** | 53 TCP/UDP | AD-integrated DNS with SRV records |
| **Kerberos** | 88, 464 TCP/UDP | Authentication and password change |
| **RPC** | 135 TCP | Endpoint Mapper for AD services |
| **NetBIOS** | 137-138 UDP, 139 TCP | Legacy name resolution and sessions |
| **LDAP** | 389 TCP/UDP | Directory queries |
| **LDAPS** | 636 TCP | LDAP over TLS |
| **SMB** | 445 TCP | File shares and IPC |
| **Global Catalog** | 3268, 3269 TCP | Forest-wide queries |
| **phpLDAPadmin** | 8090 TCP | Web GUI for browsing the directory |

Plus optional **test data**: users, groups (including nested groups), OUs, and a service account.

## Quick start

```bash
git clone https://github.com/patrickblattner/samba4-server.git
cd samba4-server
./setup.sh
```

The setup script asks you for:
- **Domain name** (e.g. `lab.dev`)
- **NetBIOS name** (e.g. `LAB`)
- **Admin password**
- **Binding mode** — internal only (`127.0.0.1`) or external (`0.0.0.0`)
- **phpLDAPadmin port** (default: `8090`)
- **Docker subnet** (default: `172.20.0.0/24`)
- **Test data** — whether to seed users, groups, and OUs

First start takes ~60-90 seconds for Samba to provision the domain.

## Useful commands

```bash
# List users
docker exec samba-ad samba-tool user list

# List groups
docker exec samba-ad samba-tool group list

# Show group members
docker exec samba-ad samba-tool group listmembers Engineers

# DNS query
docker exec samba-ad samba-tool dns query dc1 lab.dev @ ALL

# LDAP search from host
ldapsearch -x -H ldap://127.0.0.1 -D "CN=Administrator,CN=Users,DC=lab,DC=dev" -w "Admin1234!" -b "DC=lab,DC=dev" "(objectClass=user)" sAMAccountName

# Kerberos ticket (requires krb5-user)
kinit Administrator@LAB.DEV

# SMB share listing
smbclient -L //127.0.0.1 -U Administrator%Admin1234!
```

## Teardown

```bash
./teardown.sh
```

Removes all containers, volumes, and generated files in `deploy/`.

## Requirements

- Docker Desktop (or Docker Engine with Compose v2)
- ~512 MB RAM for the Samba container
- Ports listed above must be free on the chosen bind address

## How it works

- Uses the [`diegogslomp/samba-ad-dc`](https://hub.docker.com/r/diegogslomp/samba-ad-dc) Docker image
- Samba provisions a new AD domain on first start
- A separate seed container (same image) creates test data via `samba-tool` and `ldapadd`
- `phpLDAPadmin` provides a web GUI for browsing and editing the directory
- All generated config (including passwords) stays in `deploy/` which is gitignored
